========================================================================
  DÉPLOIEMENT DES SITES DU GROUPE NGF SUR cPanel (o2switch)
========================================================================

Tu as 6 fichiers ZIP, un par site. Chacun se dépose dans le "document root"
(dossier racine) de son domaine ou sous-domaine.

  ngf-groupe.com.zip            -> domaine principal ngf-groupe.com
  shop.ngf-groupe.com.zip       -> sous-domaine  shop.ngf-groupe.com
  piriou.ngf-groupe.com.zip     -> sous-domaine  piriou.ngf-groupe.com
  dees.ngf-groupe.com.zip       -> sous-domaine  dees.ngf-groupe.com
  transit.ngf-groupe.com.zip    -> sous-domaine  transit.ngf-groupe.com
  coutellerie.ngf-groupe.com.zip-> sous-domaine  coutellerie.ngf-groupe.com


ÉTAPES DANS cPanel (les mêmes pour chaque ZIP)
------------------------------------------------------------------------
1. cPanel -> "Gestionnaire de fichiers" (File Manager).

2. Ouvrir le dossier racine du site concerné :
   - ngf-groupe.com          -> dossier  public_html
   - un sous-domaine (shop…) -> le dossier choisi à sa création,
     en général  public_html/shop ,  public_html/piriou , etc.
     (Si un doute : cPanel -> "Sous-domaines", la colonne
      "Racine du document" donne le chemin exact de chacun.)

3. Vider l'ancien contenu du dossier (s'il y en a un), pour repartir propre.

4. Bouton "Téléverser" (Upload) -> envoyer le ZIP du site.

5. De retour dans le dossier, clic droit sur le ZIP -> "Extraire" (Extract).
   Les fichiers se placent directement dans le dossier racine.

6. Supprimer le fichier ZIP une fois extrait.

7. Ouvrir le site dans le navigateur pour vérifier.


IMPORTANT : les formulaires de contact et de devis (.php)
------------------------------------------------------------------------
Ils fonctionnent avec la fonction mail() de o2switch, rien à configurer.
Les demandes arrivent sur :
  - Devis (site principal) et DEES / Piriou : depcom@ngomfreres.com
  - NGF Transit                             : ngftransit@orange.sn

Seule réserve : comme l'e-mail part depuis le serveur ngf-groupe.com mais
avec une adresse @ngomfreres.com / @orange.sn, il PEUT tomber dans les
indésirables du destinataire. Si ça arrive, regarde le dossier Spam, et au
besoin l'hébergeur peut régler l'authentification (SPF) pour ces adresses.


Ces ZIP sont régénérés à chaque "python3 build.py". Ne pas modifier leur
contenu à la main : éditer les sources dans ngf-web/ puis relancer le build.
